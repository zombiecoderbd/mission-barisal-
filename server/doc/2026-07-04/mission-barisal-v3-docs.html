🔄 কাজের প্রক্রিয়া (মিশন পাইপলাইন)

ব্যবহারকারী একটি প্রশ্ন/কাজ দিলে ৩টি ফেজে পুরো প্রক্রিয়া সম্পন্ন হয়:

📥 User Input → 🔍 Classify Input → 🤖 Select Agents → ⚡ Phase 1: Parallel Response → 🔄 Phase 2: Cross-Check Debate → 📊 Phase 3: Combined Output → 📤 Final Response

ফেজ ১: সমান্তরাল রেসপন্স

সিলেক্টেড এজেন্টরা একই সময়ে কাজ শুরু করে। প্রত্যেকে নিজ নিজ রোল অনুযায়ী ব্যবহারকারীর প্রশ্নের উত্তর তৈরি করে। যাদের মডেল支持下 reasoning_content (মিমো, নর্থ মিনি, নিমোট্রন) — হক মওলা নরমালাইজার সেটা ডিটেক্ট করে reasoning থেকে প্রকৃত উত্তর এক্সট্র্যাক্ট করে।

ফেজ ২: ক্রস-চেক ও ডিবেট

স্টেপ ১: ইন্টেন্ট এক্সট্র্যাক্ট

একটি এজেন্ট ব্যবহারকারীর ইনপুট থেকে ইন্টেন্ট বের করে — কোন ভাষায়, কী জটিলতা, কোড দরকার কিনা।

স্টেপ ২: এলাইনমেন্ট চেক

প্রত্যেক এজেন্টের উত্তর যাচাই করা হয় — ব্যবহারকারীর আসল প্রশ্নের সাথে মিলছে কিনা, প্রমাণ আছে কিনা।

স্টেপ ৩: প্রুফ চেক

প্রত্যেক দাবির সাথে প্রমাণ আছে কিনা চেক করা হয়। ফাইল নাম, লাইন নম্বর, বা ওয়েব সার্চ রেজাল্ট — কিছু না থাকলে FAIL।

স্টেপ ৪: ডিবেট

প্রত্যেক এজেন্ট অন্য এজেন্টদের উত্তর ক্রস-চেক করে। একমত না হলে ডিবেট শুরু হয়।

স্টেপ ৫: চ্যালেঞ্জ রেজল্যুশন

যেসব এজেন্টের উত্তর FAIL বা DISAGREE পেয়েছে, তারা নিজেদের উত্তর সংশোধনের সুযোগ পায়।

ফেজ ৩: কম্বাইন্ড আউটপুট

কিউএ টাইরান্ট (মজনু) সব এজেন্টের উত্তর এবং ডিবেট লগ একত্রিত করে একটি ফাইনাল কম্বাইন্ড রেসপন্স তৈরি করে। যদি কোড সংক্রান্ত প্রশ্ন হয়, তাহলে কোড সেফটি রুলস অটো-ইনজেক্ট হয়।

🔌 API এন্ডপয়েন্টস

সার্ভার পোর্ট 3000-এ চলে এবং নিচের এন্ডপয়েন্টগুলো সরবরাহ করে:

GET /health হেলথ চেক — সার্ভার চলছে কিনা যাচাই
GET /identity সিস্টেম আইডেন্টিটি — ZombieCoder প্ল্যাটফর্মের তথ্য
GET /v1/models মাস্কড মডেল লিস্ট — ক্লায়েন্টদের জন্য (যেমন model-pro, model-ultra)
GET /api/v1/models আনমাস্কড মডেল লিস্ট — ডেভেলপারদের জন্য (সব প্রোভাইডারের আসল মডেল)
POST /v1/chat/completions প্রক্সি চ্যাট কমপ্লিশন — এজেন্ট বা মিশন
POST /api/mission ফুল ডিবেট মিশন — সব এজেন্টের পূর্ণাঙ্গ ডিবেট
POST /api/mcp-clients কানেক্টেড MCP ক্লায়েন্ট লিস্ট
POST /api/set-working-dir ওয়ার্কিং ডিরেক্টরি সেট করুন (zombieBridge-এর জন্য)
POST /api/normalize হক মওলা নরমালাইজার টেস্ট (ডেভ)
GET /api/normalize-list প্রোভাইডার + মডেল লিস্ট
GET /api/config রানটাইম কনফিগ দেখুন
POST /api/config রানটাইম কনফিগ আপডেট করুন
POST /mcp MCP JSON-RPC 2.0 এন্ডপয়েন্ট
WS / WebSocket রিয়েল-টাইম সংযোগ

🧟 হক মওলা নরমালাইজার

এটি একটি ইউনিভার্সাল রেসপন্স নরমালাইজার। যখন কোনো প্রোভাইডার (OpenCode, Groq, Gemini) থেকে রেসপন্স আসে, হক মওলা সেটাকে OpenAI স্ট্যান্ডার্ড ফরম্যাটে কনভার্ট করে। বিশেষ করে:

  • Mimo, North Mini, Nemotron-এর reasoning_content থেকে উত্তর এক্সট্র্যাক্ট করে
  • Anthropic ও Gemini ফরম্যাট OpenAI-তে কনভার্ট করে
  • প্রোভাইডার ডিটেক্ট করে (Groq, OpenCode, Xiaomi, ইত্যাদি)

📁 অটোমেটিক SSOT (Single Source of Truth)

যখন zombieBridge-এর মাধ্যমে set_working_dir কল করা হয়, সার্ভার সেই ডিরেক্টরি স্ক্যান করে স্বয়ংক্রিয়ভাবে .zombiecoder/SSOT.md জেনারেট করে। এতে প্রজেক্টের টেক স্ট্যাক, ফাইল কাউন্ট, ফ্রেমওয়ার্ক ডিটেকশন — সব কিছু থাকে। এজেন্টরা এই SSOT-কেই ট্রুথ হিসেবে ব্যবহার করে।

🔍 ইনপুট ক্লাসিফিকেশন

প্রতিটি ইনপুট প্রথমে ক্লাসিফাই করা হয় — Greeting, Simple QA, Project Task, Code Change, Debug Request, অথবা General Task। এর উপর ভিত্তি করে উপযুক্ত এজেন্ট সিলেক্ট করা হয়। গ্রিটিং এর জন্য মাত্র ১ জন এজেন্ট, কিন্তু কোড চেঞ্জের জন্য ৬ জন এজেন্টই কাজ করে।

📖 কিভাবে ব্যবহার করবেন

সার্ভার স্টার্ট

# ডিফল্ট পোর্ট 3000
node z.js

# কাস্টম পোর্ট

সিম্পল চ্যাট (Proxy Mode)

curl -X POST http://localhost:3000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "model-pro",
    "messages": [
      {"role": "user", "content": "কীভাবে একটি Node.js সার্ভার তৈরি করব?"}
    ]
  }'

ফুল ডিবেট মিশন

curl -X POST http://localhost:3000/api/mission \
  -H "Content-Type: application/json" \
  -d '{
    "input": "আমার কোডে বাগ খুঁজে বের করো",
    "session_id": "optional-session-id",
    "stream": true
  }'

MCP (Model Context Protocol)

curl -X POST http://localhost:3000/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "set_working_dir",
    "params": {"dir": "/home/user/project"},
    "id": 1
  }'

হক মওলা নরমালাইজার টেস্ট

curl -X POST http://localhost:3000/api/normalize \
  -H "Content-Type: application/json" \
  -d '{"raw": {"choices": [{"message": {"content": "hello"}}]}, "model": "test"}'

→ কোড গুরু + কিউএ টাইরান্ট — ২ জন এজেন্ট কাজ করে। দ্রুত উত্তর।

🐛 বাগ ফিক্স

ব্যবহারকারী: "আমার API এ ক্রাশ হচ্ছে, কেন?"

→ বাগ হান্টার + কোড গুরু + সিকিউরিটি + পারফরম্যান্স + ডক কিং + কিউএ — সবাই কাজ করে। ডিবেট হয়, প্রমাণ চেক হয়।

🏗️ প্রজেক্ট অ্যানালাইসিস

ব্যবহারকারী: "আমার প্রজেক্টের আর্কিটেকচার বিশ্লেষণ করো"

→ SSOT স্ক্যান হয়, এস enema কোড গুরু + ডক কিং + কিউএ কাজ করে।

🔒 সিকিউরিটি অডিট

ব্যবহারকারী: "আমার কোডে সিকিউরিটি ইস্যু খুঁজে বের করো"

→ নিরাপত্তা বীর বাবলু লিড করে, ওয়েব সার্চ করে লেটেস্ট ভালনারেবিলিটি চেক করে।

⚡ পারফরম্যান্স ইস্যু

ব্যবহারকারী: "আমার অ্যাপ্লিকেশন ধীর, কি করব?"

→ পারফরম্যান্স উইজার্ড রাশেদ মেমরি ও ক্যাশিং অ্যানালাইসিস করে, ওয়েব সার্চ থেকে বেঞ্চমার্ক আনে।

🤔 ব্যবহারের সময় কী কী সমস্যা হতে পারে?

❌ কোন এজেন্ট লোড হয়নি: PERSONAS.md ফাইল না থাকলে বা কোরাপ্ট হলে। সমাধান: GIT_PERSONAS_URL এনভায়রনমেন্ট ভেরিয়েবল সেট করুন অথবা PERSONAS.md ফাইল ঠিক করুন।
❌ API কী না থাকা: OpenCode, Groq বা Gemini-র API কী সেট না থাকলে। সমাধান: এনভায়রনমেন্ট ভেরিয়েবল সেট করুন (OPENCODE_API_KEY, GROQ_API_KEY, GEMINI_API_KEY)।
⚠️ নেটওয়ার্ক টাইমআউট: ওয়েব সার্চ বা API কল টাইমআউট (60s) পার করলে। স্বাভাবিক — আবার চেষ্টা করুন।
⚠️ সেশন মেমরি ওভারলোড: প্রতি সেশনে সর্বোচ্চ ৫০টি মেসেজ রাখা হয়। পুরনো মেসেজ অটো-ডিলিট হয়।
✅ সবকিছু ঠিক থাকলে: ব্যবহারকারী একটি কম্বাইন্ড আউটপুট পাবেন — যেখানে সব এজেন্টের উত্তর, ক্রস-চেক রিপোর্ট এবং ডিবেট লগ একসাথে থাকে। সব দাবির সাথে প্রমাণ (ফাইল রেফারেন্স, লাইন নম্বর, ওয়েব সার্চ ফলাফল) থাকে।

📋 SSOT — Single Source of Truth

SSOT (Single Source of Truth) হল Mission Barisal-এর সবচেয়ে গুরুত্বপূর্ণ ফিচার। যখনই zombieBridge একটি প্রজেক্ট ওপেন করে, সার্ভার সেই প্রজেক্ট স্ক্যান করে .zombiecoder/SSOT.md জেনারেট করে। এই ফাইলটিতে থাকে:

  • প্রজেক্টের নাম, রুট, টাইপ (Node.js, PHP, Python ইত্যাদি)
  • ফ্রেমওয়ার্ক ডিটেকশন (Next.js, React, Laravel ইত্যাদি)
  • সব ফাইল এবং ডিরেক্টরির তালিকা
  • এজেন্টদের জন্য নির্দেশনা — কীভাবে SSOT ব্যবহার করতে হবে

এজেন্টরা SSOT-কে ট্রুথ হিসেবে মানে। যদি কোনো তথ্য SSOT-তে না থাকে, এজেন্ট বলে: "এই তথ্য বর্তমানে SSOT এ নেই"

💡 SSOT অটো-জেনারেট হয়: zombieBridge যখন set_working_dir কল করে, সার্ভার স্বয়ংক্রিয়ভাবে SSOT জেনারেট করে। কোনো manual কাজের প্রয়োজন নেই!
